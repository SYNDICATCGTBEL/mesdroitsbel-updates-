Codes retour de l’installateur Windows

Application : MES DROITS BEL
Installateur : MES_DROITS_BEL_SALARIE_INSTALLATEUR
Type d’installateur : NSIS / Electron Builder
Dernière mise à jour : 21 juin 2026

Commande d’installation silencieuse

L’installation silencieuse utilise le commutateur NSIS standard : /S

Codes retour documentés

Code retour Signification Action recommandée
0 Installation terminée avec succès. Aucune action nécessaire.
1602 Installation annulée par l’utilisateur. Relancer l’installation si l’annulation n’était pas volontaire.
1618 Une autre installation est déjà en cours. Terminer l’autre installation Windows, puis relancer.
1638 Une version de l’application existe déjà sur l’appareil. Désinstaller l’ancienne version ou lancer la mise à jour.
112 Espace disque insuffisant. Libérer de l’espace disque, puis relancer l’installation.
12029 Échec réseau pendant une opération de téléchargement ou de connexion. Vérifier la connexion réseau et réessayer.
1625 Installation refusée par une stratégie de sécurité Windows. Vérifier la stratégie de sécurité ou les restrictions administrateur.
3010 Installation réussie, mais redémarrage Windows requis pour finaliser l’opération. Redémarrer Windows. Le programme d’installation actuel ne devrait normalement pas produire ce code, car l’application ne nécessite pas de redémarrage.
1603 Échec fatal ou erreur d’installation non classée. Relancer l’installation après fermeture de l’application, vérifier les droits Windows de l’utilisateur et l’espace disque disponible.

Codes non personnalisés

Cette table utilise des codes retour Windows standards distincts pour permettre la classification demandée par Microsoft Store. Tout code retour non nul non listé ci-dessus doit être traité comme un échec d’installation.

Documentation technique

L’installateur est basé sur NSIS. La documentation NSIS indique que le commutateur /S exécute l’installateur ou le désinstallateur en mode silencieux : documentation NSIS - Command Line Usage.

NSIS permet à un installateur de définir son code retour via SetErrorLevel : documentation NSIS - SetErrorLevel.
